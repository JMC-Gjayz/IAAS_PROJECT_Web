Application Security

Application security refers to security measures implemented at the application level to prevent data or code contained within the app from being stolen or hijacked. It includes security considerations during application development and design, as well as systems and approaches to protect apps after they are deployed. Application security may include hardware, software, and procedures for identifying and mitigating security flaws. A router that prevents Internet users from viewing a computer's IP address is an example of hardware application security. However, application-level security measures, such as an application firewall that strictly defines what activities are allowed and prohibited, are typically built into the software. Procedures can include things like an application security routine with protocols like regular testing.

Authentication

  1. is used by a server when the server needs to know exactly who is accessing their information or site.
  2. Authentication is used by a client when the client needs to know that the server is system it claims to be.
  3. Authentication by a client usually involves the server giving a certificate to the client in which a trusted third party such as Verisign or Thawte states that the server belongs to the entity (such as a bank) that the client expects it to.
  4. Authentication does not determine what tasks the individual can do or what files the individual can see. Authentication merely identifies and verifies who the person or system is.
  5. Usually, authentication by a server entails the use of a user name and password. Other ways to authenticate can be through cards, retina scans, voice recognition, and fingerprints.

Authorization

  1. Authorization is a process by which a server determines if the client has permission to use a resource or access a file.
  2. Authorization is usually coupled with authentication so that the server has some concept of who the client is that is requesting access.
  3. The type of authentication required for authorization may vary; passwords may be required in some cases but not in others.
  4. In some cases, there is no authorization; any user may use a resource or access a file simply by asking for it. Most of the web pages on the Internet require no authentication or authorization.

Encryption

  1. Encryption involves the process of transforming data so that it is unreadable by anyone who does not have a decryption key.
  2. The Secure Shell (SSH) and Socket Layer (SSL) protocols are usually used in encryption processes.
  3. All data in SSL transactions is encrypted between the client (browser) and the server (web server) before the data is transferred between the two.
  1. All data in SSH sessions is encrypted between the client and the server when communicating at the shell.
  2. By encrypting the data exchanged between the client and server, information like social security numbers, credit card numbers, and home addresses can be sent over the Internet with less risk of being intercepted during transit.
